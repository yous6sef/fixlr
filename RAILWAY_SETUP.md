# 🚀 Deployment Guide - Railway.app

This guide will help you deploy the **Flix Marketplace** to Railway.

## **Step 1: Create Railway Account**

1. Go to [railway.app](https://railway.app)
2. Sign up with your GitHub account
3. Authorize Railway to access your repositories

## **Step 2: Deploy from GitHub**

1. Click **"New Project"**
2. Select **"Deploy from GitHub Repo"**
3. Find and select **`yous6sef/fixlr`**
4. Railway will auto-detect and start deployment

## **Step 3: Add PostgreSQL Database**

1. In your Railway project dashboard, click **"+ Add"**
2. Select **"Database"** → **"PostgreSQL"**
3. Railway will create a PostgreSQL instance automatically
4. It will generate `DATABASE_URL` environment variable

## **Step 4: Configure Environment Variables**

Go to **Variables** tab and set:

### **Database**
- `DATABASE_URL` - Auto-generated by Railway PostgreSQL add-on

### **Application**
```
NODE_ENV=production
PORT=8080
HOST=0.0.0.0
```

### **Authentication (Update These)**
```
JWT_SECRET=your-production-jwt-secret-key-change-this
```

### **API Keys (Update These)**
```
HERE_API_KEY=your-here-api-key
STRIPE_SECRET_KEY=sk_live_your_stripe_key
GOOGLE_AI_KEY=your-google-ai-api-key
```

### **Email Service (Update These)**
```
EMAIL_API_KEY=your-sendgrid-api-key
EMAIL_FROM=noreply@flixmarketplace.com
```

## **Step 5: Set Build & Deploy Settings**

### **Build Command**
```bash
cd flix && composer install
```

### **Start Command**
```bash
cd flix && php -S 0.0.0.0:$PORT
```

## **Step 6: Deploy**

1. Railway will automatically deploy when you push to GitHub
2. Watch the deployment logs in the dashboard
3. Once deployed, click on your project to get your public URL

## **Step 7: Access Your Application**

Your live URL will be displayed in the Railway dashboard:
- **Format**: `https://fixlr-production.up.railway.app`
- **Landing Page**: `https://fixlr-production.up.railway.app/landing.html`
- **Login**: `https://fixlr-production.up.railway.app/login.php`
- **Signup**: `https://fixlr-production.up.railway.app/signup.php`

## **Step 8: Database Migrations**

Once deployed, run database setup:

1. Open Railway project terminal
2. Run:
```bash
cd flix
php db_migrate.php
php db_schema.php
```

Or use Railway's shell to execute:
```bash
railway run php db_migrate.php
```

## **Troubleshooting**

### **Build Fails**
- Check logs: Railway Dashboard → "Logs" tab
- Ensure `composer.json` is in `flix/` directory
- Verify PHP version is 8.1+

### **Database Connection Error**
- Verify `DATABASE_URL` is set in variables
- Check PostgreSQL database is running
- Ensure `db.php` reads `DATABASE_URL` environment variable

### **Port Issues**
- Railway automatically assigns a port
- Don't hardcode port, use `$_SERVER['SERVER_PORT']` or `$PORT` env var

### **Accessing Database from Shell**
```bash
railway run psql $DATABASE_URL
```

## **Update Database Connection in db.php**

Make sure `flix/db.php` reads environment variables:

```php
<?php
$database_url = $_ENV['DATABASE_URL'] ?? getenv('DATABASE_URL');

if ($database_url) {
    $url = parse_url($database_url);
    $host = $url['host'];
    $port = $url['port'] ?? 5432;
    $dbname = ltrim($url['path'], '/');
    $user = $url['user'];
    $password = $url['pass'];
} else {
    // Fallback to .env values
    $host = $_ENV['DB_HOST'] ?? 'localhost';
    $port = $_ENV['DB_PORT'] ?? '5432';
    $dbname = $_ENV['DB_NAME'] ?? 'flix';
    $user = $_ENV['DB_USER'] ?? 'postgres';
    $password = $_ENV['DB_PASS'] ?? '';
}

try {
    $conn = new PDO("pgsql:host=$host;port=$port;dbname=$dbname", $user, $password, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION
    ]);
} catch (PDOException $e) {
    echo "Connection failed: " . $e->getMessage();
}
?>
```

## **Add Custom Domain (Optional)**

1. In Railway project, go to **"Settings"**
2. Under **"Domains"**, click **"Add Custom Domain"**
3. Enter your domain (e.g., `fixlr.yourdomain.com`)
4. Update DNS records as instructed
5. SSL certificate is auto-generated
